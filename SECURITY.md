# Security policy

## Supported version

Security fixes apply to the latest published Failset sampler release. The runner is local-only and has no network client, account system, telemetry, or credential handling.

## Report privately

Use [GitHub private vulnerability reporting](https://github.com/armantark/failset/security/advisories/new) for a vulnerability in the runner, packaging, release workflow, or public site assets. Do not open a public issue for a security-sensitive report.

Never attach real prompts, model outputs, provider credentials, customer data, production logs, private source documents, or employer material. Reproduce the issue with a minimal synthetic fixture. Failset does not operate a bug bounty or promise a response-time SLA.

Corrections to the behavior or provenance of an existing public case that do not expose a vulnerability can use the normal contribution process.
